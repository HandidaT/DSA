Everything in js(similar to c and maybe other languages as well) happens inside an execution context(stack memory), which is created when js program file runs. This context is like a container and it has two components, which are executed separately in two phases, 1)Memory/variable environment-where all funcs and vars are stored as key-value pairs. During execution phase, each var is assigned undefined(keyword maybe like NULL) at beginning. 2)Code/thread of execution-Where code is executed a line at a time. During code execution phase vars are assigned values given by coder(static)/logic. Each func creates sub-context with memory and code((assigns args,vars,return value...)) components and phases. Context/stack(stack ds) maintains order of execution of contexts. | JS is synchronous single-threaded language(but, in contrast, AJAX is asychronous js).

FndaPDO refers to PHP Data Object. lightweight and consistent interface for accessing a database in PHP. focuses on data access abstraction rather than database abstraction--regardless of which database we are using, we have to use the same functions. PDO extension can access any database which is written for PDO driver unlike mysql_i which is mysql specific. ex:$dbConn= new PDO("mysql:host=$dbHost;dbname=$dbName",$dbUser,$dbPassword); PDO n mysql_i support Prepared Statements. Prepared Statements protect from SQL injection, and are very important for web application security. Prepared statement is SQL statement template. execute the similar SQL statements repeatedly with high efficiency. database parses, compiles, and performs query optimization on the SQL statement template, and stores the result without executing it.
